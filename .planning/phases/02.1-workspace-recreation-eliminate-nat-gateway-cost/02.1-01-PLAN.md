---

## 5. Workspace Recreation (NAT Gateway Cost Trap)

As of ARM API version 2024-05-01, Azure Databricks defaults to `enableNoPublicIp=true` when creating workspaces. This setting automatically provisions a NAT gateway in the managed resource group to handle outbound traffic from worker nodes — even for dev workspaces with no traffic. The NAT gateway costs approximately R$35/month (Azure East US pricing, Q1 2026) when completely idle.

**Why you cannot remove it manually:** Azure applies deny assignments to managed resource groups for Databricks. Even as subscription owner, `az group delete` on the managed resource group will fail with `AuthorizationFailed`. Only the Databricks service (triggered by workspace deletion) can remove those deny assignments.

### If you hit the NAT gateway cost trap

**Step 1: Delete the workspace (single command)**

```bash
az databricks workspace delete \
  --name <WORKSPACE_NAME> \
  --resource-group <WORKSPACE_RG_NAME> \
  --force-deletion \
  --yes
```

The `--force-deletion` flag removes the deny assignment and deletes the managed resource group (including the NAT gateway, VNet, and storage account). Without `--force-deletion`, the managed RG is retained and costs continue.

**Step 2: Recreate with public IPs**

```bash
az databricks workspace create \
  --name <WORKSPACE_NAME> \
  --resource-group <WORKSPACE_RG_NAME> \
  --location eastus \
  --sku premium \
  --enable-no-public-ip false \
  --managed-resource-group <MANAGED_RG_NAME>
```

Key flags:
- `--sku premium`: Required as of April 1, 2026 — Standard SKU creation is blocked in all Azure subscriptions (all existing Standard workspaces will be auto-upgraded October 1, 2026).
- `--enable-no-public-ip false`: Explicit override of the 2024-05-01 default. Worker nodes get public IPs — Azure does not provision a NAT gateway.

**Step 3: Rebuild runtime state**

After workspace recreation, the following must be restored (Entra ID and OIDC federation policies survive deletion):

1. Re-add SP as workspace admin via Databricks Account Console (UI) — see Section 1 Step 7
2. Recreate secret scope: `databricks secrets create-scope lol-pipeline`
3. Restore API key: `databricks secrets put-secret lol-pipeline riot-api-key --string-value "<key>"`
4. Grant SP READ: `databricks secrets put-acl lol-pipeline "<sp-uuid>" READ`
5. Update `DATABRICKS_HOST` GitHub variable to the new workspace URL
6. Push to `main` to trigger `cd-dev.yml` and redeploy the bundle

**Verify no NAT gateway after creation:**

```bash
az resource list \
  --resource-group <MANAGED_RG_NAME> \
  --query "[?type=='Microsoft.Network/natGateways'].name" \
  --output tsv
# Expected: empty output
```
```
